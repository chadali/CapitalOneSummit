Price / Optimization

Unfortunately due to school restrictions, I don't have time to complete price optimization, but I can give my thought process. (BONUS bottom right of screen)

Price Estimation

After calculating the average price of each neighbourhood (button below), the main difficulty is converting a long/lat value to a specific neighbourhood. If I had the time, I would try to create a "boundry" of houses that enclose a specific neighbourhood using the given data. Then, I could check if the inputted values are within any of the neighbourhood boundaries, if so I'd return the average price of that neighbourhood. If not, find the closest house to the given value and assume both houses are in the same neighbourhood.

Price Optimization

Similarly the hard work is converting a long/lat value to a neighbourhood. Once that's complete, to properly optimize I would calculate the average Square-Feet, bedrooms, baths, price of houses in that neighbourhood. Using that value, if a new house is above the average in the number of bedrooms and baths, I would return a value that's greater than the average price, and vice-versa. The key is that there's a huge amount of data (columns) which should ideally be looked at and computed.
